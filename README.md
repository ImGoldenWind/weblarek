# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Тип ApiPostMethods

```typescript
type ApiPostMethods = 'POST' | 'PUT' | 'DELETE';
```

Перечисляет допустимые HTTP-методы для отправки данных. Используется в параметре `method` метода `post` класса `Api`.

#### Интерфейс IApi

Описывает программный контракт для HTTP-клиента. Позволяет подменять реализацию транспортного уровня без изменения бизнес-логики.

```typescript
interface IApi {
    get<T extends object>(uri: string): Promise<T>;
    post<T extends object>(uri: string, data: object, method?: ApiPostMethods): Promise<T>;
}
```

`get<T>(uri: string): Promise<T>` — выполняет GET-запрос на переданный эндпоинт и возвращает промис с типизированным ответом.  
`post<T>(uri: string, data: object, method?: ApiPostMethods): Promise<T>` — выполняет запрос с телом (по умолчанию `POST`) и возвращает промис с типизированным ответом.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`off(eventName: EventName, callback: Function): void` - снимает конкретный обработчик с события.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`onAll(callback: (event: { eventName: string, data: unknown }) => void): void` - подписывается на все события; обработчик получает объект с именем события и переданными данными.  
`offAll(): void` - снимает все зарегистрированные обработчики.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

В приложении используются две основные сущности, описывающие предметную область: товар и покупатель.

### Интерфейс IProduct

Описывает структуру данных одного товара в магазине.

```typescript
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```

Поля:  
`id: string` — уникальный идентификатор товара, используется для его поиска и идентификации в корзине.  
`description: string` — текстовое описание товара, отображается в модальном окне с деталями.  
`image: string` — URL изображения товара.  
`title: string` — название товара.  
`category: string` — категория, к которой относится товар.  
`price: number | null` — цена товара в синапсах; `null` означает, что товар бесценен и не может быть добавлен в корзину.

### Интерфейс IBuyer

Описывает данные покупателя, которые тот вводит при оформлении заказа.

```typescript
type TPayment = 'online' | 'cash';

interface IBuyer {
  payment: TPayment | '';
  email: string;
  phone: string;
  address: string;
}
```

Поля:  
`payment: TPayment | ''` — выбранный способ оплаты: `'online'` (онлайн) или `'cash'` (при получении). Хранит пустую строку, пока способ оплаты не выбран.  
`email: string` — электронная почта покупателя.  
`phone: string` — номер телефона покупателя.  
`address: string` — адрес доставки.

## Модели данных

Работа с данными в приложении организована через три класса-модели, каждый из которых отвечает за свою зону ответственности.

### Класс ProductCatalog

Отвечает за хранение и управление списком всех товаров, доступных в магазине, а также за хранение товара, выбранного пользователем для просмотра подробной информации.

Конструктор:  
`constructor(events: IEvents)` — принимает брокер событий, через который модель оповещает приложение об изменении своего состояния.

Поля класса:  
`_items: IProduct[]` — массив всех товаров, загруженных с сервера.  
`_preview: IProduct | null` — товар, выбранный для подробного отображения в модальном окне. Хранит `null`, если ни один товар не выбран.

Методы класса:  
`setItems(items: IProduct[]): void` — принимает массив товаров и сохраняет его в модели. Вызывается после получения данных с сервера.  
`getItems(): IProduct[]` — возвращает массив всех товаров каталога.  
`getItem(id: string): IProduct | undefined` — принимает идентификатор товара и возвращает соответствующий объект товара из массива, либо `undefined`, если товар не найден.  
`setPreview(item: IProduct): void` — принимает товар и сохраняет его как выбранный для подробного отображения.  
`getPreview(): IProduct | null` — возвращает товар, выбранный для подробного отображения, или `null`.

---

### Класс Basket

Отвечает за хранение и управление набором товаров, которые пользователь выбрал для покупки.

Конструктор:  
`constructor(events: IEvents)` — принимает брокер событий, через который модель оповещает приложение об изменении состава корзины.

Поля класса:  
`_items: IProduct[]` — массив товаров, добавленных пользователем в корзину.

Методы класса:  
`getItems(): IProduct[]` — возвращает массив товаров, находящихся в корзине.  
`addItem(item: IProduct): void` — принимает объект товара и добавляет его в корзину.  
`removeItem(item: IProduct): void` — принимает объект товара и удаляет его из корзины.  
`clear(): void` — очищает корзину, удаляя все товары.  
`getTotalPrice(): number` — возвращает суммарную стоимость всех товаров в корзине.  
`getCount(): number` — возвращает количество товаров в корзине.  
`hasItem(id: string): boolean` — принимает идентификатор товара и возвращает `true`, если товар с таким `id` присутствует в корзине, иначе `false`.

### Класс BuyerData

Отвечает за хранение и валидацию данных покупателя, которые тот вводит в процессе оформления заказа.

Конструктор:  
`constructor(events: IEvents)` — принимает брокер событий, через который модель оповещает приложение об изменении данных покупателя.

Поля класса:  
`_payment: TPayment | ''` — выбранный способ оплаты. Хранит пустую строку, пока способ не выбран.  
`_address: string` — адрес доставки.  
`_phone: string` — номер телефона покупателя.  
`_email: string` — электронная почта покупателя.

Методы класса:  
`setField(data: Partial<IBuyer>): void` — принимает объект с частичными данными покупателя и обновляет только те поля модели, которые присутствуют в переданном объекте. Поля, отсутствующие в аргументе, остаются без изменений.  
`getData(): IBuyer` — возвращает объект со всеми данными покупателя в виде, соответствующем интерфейсу `IBuyer`.  
`clear(): void` — сбрасывает все поля покупателя в начальные пустые значения. Вызывается после успешного оформления заказа.  
`validate(): Partial<Record<keyof IBuyer, string>>` — проверяет заполненность всех полей покупателя. Возвращает объект, в котором присутствуют только поля с ошибками; значением каждого поля является текстовое описание ошибки. Если все поля заполнены, возвращает пустой объект. Поле считается валидным, если оно не пустое. Например:
```typescript
// Пример возвращаемого значения при незаполненных payment и email:
{
  payment: 'Не выбран вид оплаты',
  email: 'Укажите емэйл',
}
```

## Слой коммуникации

Отвечает за взаимодействие приложения с внешним API сервера «Web-Larёk». Классы этого слоя не хранят состояние приложения — они лишь формируют запросы и возвращают промисы с результатом.

### Типы данных для взаимодействия с сервером

```typescript
// Ответ сервера на GET /product/
interface IProductList {
    total: number;
    items: IProduct[];
}

// Тело запроса POST /order/ (расширяет IBuyer, добавляя итоговую сумму и список id товаров)
interface IOrderRequest extends IBuyer {
    total: number;
    items: string[];
}

// Ответ сервера на успешное создание заказа
interface IOrderResponse {
    id: string;
    total: number;
}
```

`IProductList` — структура ответа сервера при запросе списка товаров. Поле `total` содержит общее количество товаров в каталоге, `items` — массив объектов товаров.

`IOrderRequest` — данные, отправляемые на сервер при оформлении заказа. Расширяет `IBuyer` (способ оплаты, email, телефон, адрес), добавляя итоговую сумму заказа `total` и массив идентификаторов выбранных товаров `items`.

`IOrderResponse` — подтверждение от сервера об успешном создании заказа. Содержит `id` созданного заказа и подтверждённую сумму `total`.

### Класс WebLarekApi

Отвечает за взаимодействие с API сервера «Web-Larёk». Использует композицию: принимает в конструктор объект, реализующий интерфейс `IApi`, и делегирует ему выполнение HTTP-запросов. Сам класс не знает о деталях транспортного уровня — это позволяет легко подменять реализацию `IApi` при тестировании.

Конструктор:  
`constructor(api: IApi)` — принимает объект, реализующий интерфейс `IApi` (например, экземпляр базового класса `Api`).

Поля класса:  
`_api: IApi` — хранит ссылку на переданный объект для выполнения HTTP-запросов.

Методы класса:  
`getProducts(): Promise<IProductList>` — выполняет GET-запрос на эндпоинт `/product/` и возвращает промис с объектом `IProductList`, содержащим массив товаров и их общее количество.  
`createOrder(order: IOrderRequest): Promise<IOrderResponse>` — принимает объект данных заказа типа `IOrderRequest` и выполняет POST-запрос на эндпоинт `/order/`. Возвращает промис с объектом `IOrderResponse`, подтверждающим успешное создание заказа.

## Слой представления

Классы слоя View отвечают за отображение данных в интерфейсе. Каждый класс управляет своим блоком разметки. Классы не хранят состояние приложения и не обращаются к моделям напрямую. В ответ на действия пользователя они генерируют события, которые обрабатывает Презентер.

### Класс Page

Управляет корневым блоком страницы: галереей товаров и кнопкой корзины в шапке.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — принимает корневой элемент страницы и брокер событий.

Поля класса:  
`_gallery: HTMLElement` — контейнер `<main class="gallery">`, в который рендерятся карточки каталога.  
`_basketButton: HTMLElement` — кнопка открытия корзины в шапке.  
`_basketCounter: HTMLElement` — элемент со счётчиком товаров в корзине.

Методы класса:  
`set catalog(items: HTMLElement[]): void` — принимает массив DOM-элементов карточек и отображает их в галерее.  
`set counter(value: number): void` — обновляет отображаемое количество товаров в счётчике корзины.

События:  
`basket:open` — генерируется при клике на кнопку корзины.

### Класс Modal

Управляет модальным окном. Умеет открываться, закрываться и принимать произвольный контент. Не имеет дочерних классов — содержимое модального окна является самостоятельным компонентом, который передаётся извне.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — принимает элемент `div#modal-container` и брокер событий.

Поля класса:  
`_content: HTMLElement` — контейнер `div.modal__content`, в который помещается переданный компонент.  
`_closeButton: HTMLButtonElement` — кнопка закрытия модального окна.

Методы класса:  
`set content(value: HTMLElement): void` — помещает переданный DOM-элемент в область контента модального окна.  
`open(): void` — открывает модальное окно, добавляя активный CSS-класс.  
`close(): void` — закрывает модальное окно, удаляя активный CSS-класс и очищая контент.

События:  
`modal:open` — генерируется при открытии модального окна.  
`modal:close` — генерируется при закрытии модального окна (по кнопке или клику на оверлей).

### Интерфейс ICardActions

Описывает опциональные обработчики действий для карточки товара.

```typescript
interface ICardActions {
    onClick?: () => void;
}
```

`onClick?: () => void` — вызывается при взаимодействии пользователя с карточкой: клик по всей карточке (`CardCatalog`) или по кнопке «В корзину» (`CardPreview`).

### Класс Card

Базовый класс для всех вариантов карточки товара. Содержит общий функционал, который используется во всех трёх шаблонах карточки: управление заголовком, изображением, категорией и ценой.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает корневой элемент карточки, склонированный из соответствующего шаблона, и опциональный объект с обработчиками действий. Если передан `actions.onClick`, привязывает его к событию клика на контейнер карточки.

Поля класса:  
`_title: HTMLElement` — элемент с названием товара.  
`_price: HTMLElement` — элемент с ценой товара.  
`_image?: HTMLImageElement` — изображение товара (присутствует не во всех шаблонах).  
`_category?: HTMLElement` — элемент с категорией товара (присутствует не во всех шаблонах).

Методы класса:  
`set title(value: string): void` — устанавливает название товара.  
`set price(value: number | null): void` — устанавливает цену товара; при `null` отображает «Бесценно».  
`set image(value: string): void` — устанавливает изображение товара.  
`set category(value: string): void` — устанавливает категорию и соответствующий CSS-модификатор цвета.

### Класс CardCatalog (наследует Card)

Карточка товара для отображения в галерее главной страницы. Использует шаблон `#card-catalog`. При клике на карточку генерирует событие для открытия детального просмотра.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает элемент карточки и опциональный объект с обработчиками. Обработчик клика передаётся через `actions.onClick`.

События:  
`card:select` — генерируется при клике на карточку; передаёт данные карточки в Презентер.

### Класс CardPreview (наследует Card)

Карточка товара для детального просмотра в модальном окне. Использует шаблон `#card-preview`. Дополнительно отображает описание товара и кнопку добавления в корзину.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает элемент карточки и опциональный объект с обработчиками. `actions.onClick` привязывается к кнопке «В корзину», а не к контейнеру карточки.

Поля класса:  
`_description: HTMLElement` — элемент с описанием товара.  
`_button: HTMLButtonElement` — кнопка «В корзину» / «Убрать из корзины».

Методы класса:  
`set description(value: string): void` — устанавливает текст описания.  
`set buttonText(value: string): void` — изменяет текст кнопки в зависимости от состояния товара в корзине.  
`set buttonDisabled(value: boolean): void` — блокирует кнопку для бесценных товаров (когда `price === null`).

События:  
`card:toBasket` — генерируется при клике на кнопку; передаёт данные товара в Презентер.

### Интерфейс ICardBasketActions

Описывает опциональные обработчики действий для карточки в корзине.

```typescript
interface ICardBasketActions {
    onDelete?: () => void;
}
```

`onDelete?: () => void` — вызывается при клике на кнопку удаления товара из корзины.

### Класс CardBasket (наследует Card)

Карточка товара в списке корзины. Использует шаблон `#card-basket`. Отображает порядковый номер, название, цену и кнопку удаления.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardBasketActions)` — принимает элемент карточки и опциональный объект с обработчиками. Если передан `actions.onDelete`, привязывает его к кнопке удаления.

Поля класса:

`_deleteButton: HTMLButtonElement` — кнопка удаления товара из корзины.

Методы класса:  
`set index(value: number): void` — устанавливает порядковый номер товара.

События:  
`basket:removeItem` — генерируется при клике на кнопку удаления; передаёт идентификатор товара в Презентер.

### Класс BasketView

Отображает содержимое корзины в модальном окне. Использует шаблон `#basket`. Рендерит список товаров, общую сумму и кнопку перехода к оформлению.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — принимает корневой элемент корзины и брокер событий, устанавливает обработчик на кнопку оформления.

Поля класса:  
`_list: HTMLElement` — контейнер `ul.basket__list` для карточек товаров.  
`_totalPrice: HTMLElement` — элемент с итоговой суммой.  
`_button: HTMLButtonElement` — кнопка «Оформить».

Методы класса:  
`set items(cards: HTMLElement[]): void` — принимает массив DOM-элементов карточек корзины и рендерит их в список. Если массив пуст, кнопка «Оформить» блокируется.  
`set total(value: number): void` — устанавливает итоговую сумму.

События:  
`order:start` — генерируется при клике на кнопку «Оформить».

### Интерфейс IFormState

Описывает состояние, которое передаётся в базовую форму при рендере.

```typescript
interface IFormState {
    valid: boolean;
    errors: string;
}
```

`valid: boolean` — определяет, активна ли кнопка отправки формы.  
`errors: string` — текст ошибки валидации, отображаемый под формой.

### Класс Form

Базовый класс для всех форм оформления заказа. Содержит общий функционал: управление кнопкой отправки и отображением ошибок валидации. При вводе данных в любое поле генерирует событие для Презентера.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — принимает элемент формы и брокер событий, устанавливает обработчики на событие `input` и `submit`.

Поля класса:  
`_submitButton: HTMLButtonElement` — кнопка отправки формы.  
`_errorContainer: HTMLElement` — элемент `span.form__errors` для вывода текста ошибок.

Методы класса:  
`set valid(value: boolean): void` — включает или блокирует кнопку отправки в зависимости от валидности формы.  
`set errors(value: string): void` — отображает текст ошибки валидации под формой.

События:  
`{formName}:input` — генерируется при изменении любого поля формы. Имя события формируется динамически из атрибута `name` HTML-элемента формы (например, `order:input` или `contacts:input`). Передаёт объект `{ field: string, value: string }` с именем изменённого поля и его значением.  
`{formName}:submit` — генерируется при отправке формы. Имя события формируется аналогично (например, `order:submit` или `contacts:submit`).

### Класс OrderForm (наследует Form)

Форма первого шага оформления заказа: выбор способа оплаты и ввод адреса доставки. Использует шаблон `#order`.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — принимает элемент формы и брокер событий, устанавливает обработчики на кнопки выбора оплаты.

Поля класса:  
`_paymentButtons: HTMLButtonElement[]` — массив кнопок выбора способа оплаты («Онлайн» и «При получении»).  
`_addressInput: HTMLInputElement` — поле ввода адреса доставки.

Методы класса:  
`set payment(value: TPayment | ''): void` — визуально выделяет активный способ оплаты, снимая выделение с остального. При пустой строке все кнопки остаются неактивными.  
`set address(value: string): void` — устанавливает значение в поле адреса.

События:  
`order:paymentChange` — генерируется при клике на кнопку способа оплаты; передаёт выбранное значение.  
`order:submit` — генерируется при клике на кнопку «Далее» (наследуется от Form).

| `basket:changed` | `Basket.addItem()`, `removeItem()`, `clear()` | Состав корзины изменился |

| Событие | Источник | Передаваемые данные | Описание |

## Презентер

Роль Презентера выполняет основной скрипт приложения — файл `src/main.ts`. Выносить код в отдельный класс не требуется, так как у приложения одна страница и один сценарий взаимодействия.

Презентер создаёт экземпляры всех моделей, представлений и коммуникационного класса, передавая им общий брокер событий `EventEmitter`. Вся логика приложения реализована через обработчики событий — Презентер подписывается на события, вызывает методы моделей и представлений, но сам никогда не генерирует события.

Представления перерисовываются только в двух случаях:
- при обработке события от Модели данных (данные изменились);
- при обработке события открытия модального окна.

Для синхронизации состояния между событиями в Презентере хранятся две вспомогательные переменные:
- `currentStep: 'order' | 'contacts' | null` — текущий шаг оформления заказа, определяет какую форму обновлять при `buyer:changed`;
- `currentPreviewCard: CardPreview | null` — ссылка на текущий открытый компонент превью, позволяет обновлять текст кнопки при изменении корзины без повторного открытия модального окна.

### Поток данных приложения

1. При старте — запрос к серверу `GET /product/` → `catalog.setItems()` → `catalog:changed` → карточки отрисовываются на главной странице.
2. Клик на карточку → `card:select` → `catalog.setPreview()` → `preview:changed` → модальное окно с детальным просмотром.
3. Клик «В корзину» → `card:toBasket` → `basket.addItem()` / `basket.removeItem()` → `basket:changed` → обновление счётчика и кнопки в превью.
4. Клик на иконку корзины → `basket:open` → модальное окно с корзиной.
5. Клик «Оформить» → `order:start` → модальное окно с формой оплаты и адреса.
6. Ввод данных в форму → `order:input` / `order:paymentChange` → `buyer.setField()` → `buyer:changed` → валидация и обновление формы.
7. Клик «Далее» → `order:submit` → модальное окно с формой email и телефона.
8. Ввод контактов → `contacts:input` → `buyer.setField()` → `buyer:changed` → валидация и обновление формы.
9. Клик «Оплатить» → `contacts:submit` → `POST /order/` → очистка корзины и данных → экран успеха.
10. Клик «За новыми покупками!» → `success:close` → закрытие модального окна.
